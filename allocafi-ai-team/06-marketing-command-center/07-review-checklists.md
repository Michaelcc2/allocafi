# Marketing Review Checklists

## Marketing Asset Checklist

- The copy says AllocaFi is non-custodial.
- The copy does not call AllocaFi a bank.
- The copy does not call AllocaFi a custodial wallet.
- The copy does not imply AllocaFi holds funds.
- The copy explains a real user problem.
- The copy explains a clear AllocaFi solution.
- The copy has a specific call to action.
- The copy avoids vague hype.

## Security Messaging Checklist

- No private keys, seed phrases, or recovery phrases are requested.
- Wallet verification is explained as a signature, not a transaction.
- Any transaction language says the user approves it in their external wallet.
- Security benefits are explained without internal exploit details.
- No "unhackable" or absolute security claims are used.
- Security Agent review is required before public launch.

## Legal Core And Tax Wording Checklist

- No legal advice claim.
- No tax advice claim.
- No guaranteed tax savings.
- No guaranteed deduction.
- No guaranteed investment outcome.
- Reports are described as records, suggestions, exports, or preparation.
- User/accountant review language is present where needed.

## Launch Campaign Checklist

- Audience is defined.
- Problem is clear.
- AllocaFi feature is named.
- Value is obvious in under two seconds.
- CTA is clear.
- Security wording is safe.
- Screenshots or demos do not expose private user data.
- Product, Security, QA, and Documentation have reviewed.

## Final Approval Path

1. Marketing Agent drafts.
2. Product Agent checks positioning.
3. Security Agent checks public-safe details.
4. Legal Core or Tax Agent reviews if relevant.
5. QA Agent checks accuracy against the actual app.
6. Documentation Agent archives final copy.
